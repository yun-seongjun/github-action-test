## 작업 지시

### 검토 항목
1. 코드 스타일 가이드 준수
2. 의미있는 변수 이름 사용
3. DRY 원칙 준수 (Don't Repeat Yourself)
4. KISS 원칙 준수 (Keep It Simple, Stupid)
5. YAGNI 원칙 준수 (You Aren't Gonna Need It)

### 수행할 작업

**2단계**: 개선이 필요한 특정 코드 라인에는 인라인 댓글을 남기세요.

**3단계**: 반드시 `gh pr comment ${{ github.event.pull_request.number }} --body "..."` 명령어를 사용하여 아래 형식의 최종 요약을 PR 코멘트로 남기세요.
